                    This problem can be solved by combining the binary search with the linked list concept. The linked list has a low and high node. The add function compares the given value to the current node and if less {if low==null {low is set to the added value} else {low.add is executed}} else {ditto low logic for high} You need a balancing routine because a lot of sequential adds will cause the high list to be heavily loaded and no or few low values are loaded. To balance it has a count function that tells you how many lists are in the low and high sides. (With a huge list, use two count variables, and the count function is the sum of the two variables plus 1) You have an indexing function to traipse down the list halfway minus 1 and make the low or if null the high node the top node in a new list and set the low (or high) node to null in your traiped node and its count to 0 then add the original top node to the new node list.
                    The balancing function would only implement when the total count is greater than a certain amount and the low count is less than 40% or greater than 60% of the high node
