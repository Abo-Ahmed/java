Part 1 of this tutorial talked about the Java Code which would execute a task on a scheduled basis. On this part of the tutorial, I will talk about how to configure Tanuki Software's Java Service Wrapper so the Java Application created in the first part of this tutorial can be run as a Windows Service.

Part 2 - Configure the Java Service Wrapper

Steps:  

Download the Java Service Wrapper from Tanuki Software's website

Choose the platform. Since this tutorial is about Windows, I have downloaded the Windows x86 32-bit Community Edition package (yes, it is free to download with GPL2 license).

Extract the contents of the package.

The package is a zip file. Extract it anywhere on the your desktop and you'll be presented with the directory structure like the one below.

Build the Java Application and Export it as a Runnable JAR File

On SpringSource Tool Suite, right-click on the Project and click Export. Click Java > Runnable JAR File and hit Next.

Choose your run configuration and the location where the JAR File should reside. Choose "Copy required libraries into a sub-folder next to the generated JAR" option under Library Handling and click Finish.

 

Configure Java Service Wrapper and the Java Application

Put the generated Runnable JAR File in the bin directory of the Java Service Wrapper. The library files will then go to the lib folder of the Java Service Wrapper.

Go to the bin directory of the Java Service Wrapper. The batch files (.bat) in there can be renamed and we can delete what we do not need.

Here I have renamed some files and have deleted some.

  • DemoApp.bat --> run.bat
  • InstallTestWrapper-NT.bat --> install-service.bat
  • PauseTestWrapper-NT.bat --> deleted
  • QueryTestWrapper-NT.bat --> deleted
  • ResumeTestWrapper-NT.bat --> deleted
  • StartTestWrapper-NT.bat --> start-service.bat
  • StopTestWrapper-NT.bat --> stop-service.bat
  • TestWrapper.bat --> deleted
  • TestWrapperCommand.bat --> deleted
  • UninstallTestWrapper-NT.bat --> uninstall-service.bat

run.bat is the only file that needs to be edited. The variable WRAPPER_CONF_DEFAULT must be set to ../conf/wrapper.conf

After this, go to the conf folder of the Java Service Wrapper. There is a file in there which is named wrapper.conf. The following are the bare minimum variables need to be updated to run the Java Application as a Windows Service.

Note: You can tweak the other variables on the wrapper.conf file.

  • wrapper.java.mainclass - This variable should contain the Main class to run
  • wrapper.java.classpath.1....n - These should contain the paths of the JAR files under the lib directory and also the Runnable JAR file that is in the bin directory. Note: The path should be relative to the bin directory.
  • wrapper.name - This is the name for the Windows Service
  • wrapper.displayname - This is the display name for the Windows Service
  • wrapper.description - A brief description about the Windows Service
  • wrapper.ntservice.starttype - This is the startup type of the Windows Service. AUTO_START will start the service upon booting, DELAY_START will start the service upon logon and DEMAND_START will only start the Java Application when it is explicitly executed.

# Java Main class.  This class must implement the WrapperListener interface
#  or guarantee that the WrapperManager class is initialized.  Helper
#  classes are provided to do this for you.  See the Integration section
#  of the documentation for details.
wrapper.java.mainclass=com.sample.javawindowsservice.Main
 
# Java Classpath (include wrapper.jar)  Add class path elements as
wrapper.java.classpath.1=../lib/aopalliance-1.0.jar
wrapper.java.classpath.2=../lib/commons-logging-1.1.3.jar
wrapper.java.classpath.3=../lib/spring-aop-4.0.6.RELEASE.jar
wrapper.java.classpath.4=../lib/spring-beans-4.0.6.RELEASE.jar
wrapper.java.classpath.5=../lib/spring-context-4.0.6.RELEASE.jar
wrapper.java.classpath.6=../lib/spring-context-support-4.0.6.RELEASE.jar
wrapper.java.classpath.7=../lib/spring-core-4.0.6.RELEASE.jar
wrapper.java.classpath.8=../lib/spring-expression-4.0.6.RELEASE.jar
wrapper.java.classpath.9=../lib/wrapper.jar
wrapper.java.classpath.10=java-windows-service-sample.jar
 
# Name of the service
wrapper.name=JavaWindowsServiceSample
 
# Display name of the service
wrapper.displayname=Java Windows Service Sample
 
# Description of the service
wrapper.description=A sample java windows service application
 
# Mode in which the service is installed.  AUTO_START, DELAY_START or DEMAND_START
wrapper.ntservice.starttype=AUTO_START
